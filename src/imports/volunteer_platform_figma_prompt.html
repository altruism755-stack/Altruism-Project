Project overview

Design a complete web platform called VolunteerBridge that connects non-profit organizations with volunteers. The platform has three roles: Organization (full admin access to all data), Supervisor (manages volunteers only), and Volunteer (views their organization and manages their own activities).


Design direction

The visual language should be warm, trustworthy, and purposeful — this is a civic platform, not a SaaS tool. Use a clean white surface with generous whitespace, a single primary green accent color (#16A34A), soft neutrals for secondary surfaces, and Inter as the typeface throughout. Prioritize readability and usability over decorative elements. All layouts must be fully responsive (desktop, tablet, mobile).

Organization
Full access. Manages all data, volunteers, supervisors, events, and reports. Creates the workspace.
👤

Manages volunteers assigned to their team. Can approve activities and view volunteer progress.
🙋
Volunteer
Sees their organization, logs activities, tracks hours, and manages their own profile and history.

UX principles to follow
  • Empty states: Every list or table that could be empty must have a designed empty state — a simple centered message and a primary CTA to add the first item. Never show a blank page.
  • Loading states: Show skeleton loaders (gray rounded rect placeholders matching content shape) on tables and card grids while data loads.
  • Toast notifications: Floating toasts bottom-right for success ("Volunteer approved"), error ("Something went wrong — try again"), and info. Auto-dismiss after 4 seconds. Green left-accent for success, red for error.
  • Confirmation dialogs: Destructive actions (Remove volunteer, Delete event) must trigger a modal confirmation — centered overlay, white card, action description, "Cancel" ghost + "Confirm" red primary.
  • Role-aware navigation: The sidebar or top nav must only show links relevant to the logged-in role. Org sees all sections; supervisor sees only their team sections; volunteer sees only their personal sections.
  • Responsive behavior: On mobile, top nav collapses to a hamburger drawer. Two-column layouts stack to single column. Tables become card lists on small screens.
  • Consistent page anatomy: Every page follows the same structure — breadcrumb (if needed) → page title → optional subtitle → content. No orphaned sections floating without a header.
  • Form accessibility: Every input has a visible label (never placeholder-only), required fields are marked with a small red asterisk, error messages appear below the relevant field in red with an X icon.
